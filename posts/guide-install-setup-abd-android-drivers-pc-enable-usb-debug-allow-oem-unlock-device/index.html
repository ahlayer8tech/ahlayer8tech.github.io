1. Set Up ADB on PC

  • In this section you will only be using the PC, but will not use the command prompt.

1-1 On Windows - Admin Privilege Required

  • In this section ADB will be installed and set up to be runable from anywhere on the PC. ADB files must be manually placed in a folder and manually be added to the enviornment variable in system properties.

  1. Download the latest Platform Tools for Windows from the Android developers website: https://developer.android.com/studio/releases/platform-tools

  2. Extract the folder from the zip file.

  3. Move the extracted folder to your prefered location on your PC. In this example it will be moved to the root folder of the logged in user.

Recommended to rename the folder from “platform-tools” to “ADB-platform-tools”.

%USERPROFILE%\Downloads -> %USERPROFILE%\

  1. Open Windows Start menu, type “About PC” and press enter.
Win-key > type "about pc" + press enter = About menu

  1. Click “Advanced System Settings”. 2nd from the bottom (on right panel).
About menu > Advanced System Settings

  1. Click “Environment Variables”. At the bootom of the menu.
System Properties > Environment Variables

  1. For the bottom box labeled “System Variables”, look for the variable named “Path” and double click it, or single click it to select it and then click the “Edit” button.

Best practice would be create a user account specific for Android dev and have ADB only for that users system variables.

Environment Variables > Edit Environment Variables

  1. BEFORE you proceed, ENSURE that NO line that contains text is highlighted. Click on a blank line below the filled with text lines, click “Browse”, and search for the folder where the adb folder is located. Once the desired folder is selected, click “OK”.

  2. Verify visually that the correct folder path is now present in this current menu(“Edit Environment Variables”).

  3. To exit the rest of these menus now that the task of adding the folder path to the system variables is done; click “OK” to exit the “Enviornment Variables” menu, and then click “OK” to exit the “Advance System Settings” menu.

  4. Check to confirm that the adb and fastboot commands run.

To test this, open a command prompt and type “adb version”. If it outputs the version number that matches what you downloaded from the website in step 1, then that means that adb/platform tools has been successfully set up on the PC.

Win-key > type "cmd" + press enter = command prompt

In command prompt: type "adb version" + press enter = ADB version number output on screen

   

1-2 On Windows - NO Admin Privilege Required

  • In this section ADB will be installed in your prefered directory, but it will not runable from anywhere on the PC. ADB files must manually be placed in your prefered folder, where you would run it.

  1. Download the latest Platform Tools for Windows from the Android developers website: https://developer.android.com/studio/releases/platform-tools

  2. Extract the folder from the zip file.

  3. Move the extracted folder to your prefered location on your PC. In this example it will be moved to the root folder of the logged in user “%USERPROFILE%”, for simplicity.

Recommended to rename the folder from “platform-tools” to “ADB-platform-tools”.

%USERPROFILE%\Downloads -> %USERPROFILE%

  1. Open a command prompt inside the folder you placed the adb-platform-tools in, and type “adb version”. If it outputs the version number that matches what you downloaded from the website in step 1, then that means that adb/platform tools has been successfully set up on the PC.
In the address bar of the ADB-platform-tools folder type "cmd" + press enter = command prompt with ADB functionality

In command prompt: type "adb version" + press enter = ADB version number output on screen

  1. To automate the process of opening the command prompt with adb/platform tools, create a batch script, and the script can be launched from anywhere. Example script text below.
start cmd /K "cd %USERPROFILE%\ADB-platform-tools\"
  • Recomended to name the batch script something obvious like “ADB cmd start script.bat”

   

1-3 On Linux - Package:

  • In this section you will only be using the PC.
  • This section is only required for Linux PCs.
  • On Linux PCs USB devices cannot be used as non-root without udev rules being setup for each type of device. The Linux Android device driver comes preinstalled on Linux systems(it is apart of/in the Linux kernel).
  • Do not use the adb files from the Ubuntu and Debian package repositories because they will be very outdated, and this may cause errors due to compatability issues with some newer versions of Android.
  • As of publication of this guide the packages that will be downloaded are on version 28.0.2+6. Link to check the most recent version https://pkgs.org/download/android-sdk-platform-tools-common
  1. Open a terminal and type:
  • For Debian, Ubuntu, LinuxMint, & derivatives:
sudo apt update
sudo apt install android-sdk-platform-tools-common
  • For Fedora & derivatives:
sudo dnf install android-tools
  • For Arch, Manjaro, & derivatives:
sudo pacman -Sy
sudo pacman -S android-udev
  • The proper tools and settings have been installed and configured after the install has completed.
  1. Logout of your account, and then log back into your account. The udev rules should now have been initialized on the Linux PC.

   

1-4 On Linux - Manual Configuration:

  • This methood of installing ADB has been deprecated. The methood in section 1-3 is recommended methood of installing adb on Linux PCs.

  • In this section you will only be using the PC.

The adb files from the file repo for a OSs like Ubuntu and LinuxMint will be outdated(by accouple years), and which can cause errors due to compatability issues with newer versions of Android. It is recommended that Linux users use the methood listed below to install the latest adb files.

  1. Download the latest Platform Tools for Linux from the Android developers website: https://developer.android.com/studio/releases/platform-tools

  2. Extract the folder from the zip file.

  3. Move the extracted folder to your prefered location on your PC. In this example it will be moved to the current user’s home directory. Recomended to rename the folder from “platform-tools” to “ADB”.

Downloads -> Home directory
  1. Add the path to the PC configuration - On Ubuntu-based distros:

In the terminal, add “export PATH="$HOME/platform-tools:$PATH” or export PATH="$HOME/ADB:$PATHtext to your .bashrc file (it is recomended to add it to the very end)

Note: Run the command below in terminal while it is in the home directory.

sudo "export PATH="$HOME/platform-tools:$PATH" >> .bashrc

or

sudo "export PATH="$HOME/ADB:$PATH" >> .bashrc

  1. Logout of the PC, and then login to the PC.

  2. Confirm that the adb and fastboot commands run in the terminal.

2. Configure ADB to work on the Android device

  • In this section you will only be using both the PC and the Android device.

  • Confirm that you have completed the section above before continuing with the steps below.

2-1 Enable USB Debugging and Allow OEM Unlocking on the Android device

  • In this subsection you will only be using the device.

  1. From the homesreen of the device go the “Settings” app.

  2. In the “Settings” app go to the “About Phone” menu, located at the bottom in the “Settings” app.

Settings > About Phone

  1. In the “About Phone” menu go to the “Build Number”, located near the bottom of the menu, and tap on it 7 times to enable “Developer options”. Note: The device will have pop-up notification giving congratulation for enabling “Developer options”.
Settings > About Phone > Build Number + 7 taps

  1. Go back to the “Settings” main menu, and go to the “Developer options” menu, located near the bottom of the “Settings” app.
Settings > Developer Options

  1. In the “Developer options” menu go to enable OEM Unlock.
Settings > Developer Options > OEM Unlock "Enable"

  1. In the “Developer options” menu go to enable Android debugging.
Settings > Developer Options > Android debugging "Enable"

   

2-2 Allow USB debugging of the device by the PC

  • In this section you will be using both the PC and the Android device.

  • Confirm that you have completed the section above before continuing with the steps below.

  1. Connect the Android device to the PC, using the USB cable.

  2. With the device turned on and on the home screen, on the PC, open the command prompt (on Windows) or terminal (on Linux) with adb, and type “adb devices” and press enter:

In command prompt: type "adb devices" + press enter

  • On the device screen, Confirm/Accept the RSA key signature message that pops-up on the device’s screen. The RSA key is the key of your connected PC. It is recomended to checkmark the “Always allow from this computer” box to avoid repititons in later proceduares

  • After the RSA key of the PC is accepted, on the PC in the command prompt, the device ADB number will display. Save/write down the device number that is shown for later reference.

3. Install Android Device Driver

  • In this section you will be using the PC and the Android device.
  • This section is only required for Windows PCs. The Linux Android device driver comes preinstalled on Linux systems(it is apart of/in the Linux kernel).
  • This section is critical, because it affects fastboot commands functionality (on Windows PCs).

3-1 Install Android Device Driver - Manually

  1. Before proceeding with the steps below start with section 2-2 above, and then proceed with the steps below.

  2. Make sure the device is connected to the PC, turned on, and on the home screen; in the PC open the command prompt with adb and type “adb reboot bootloader” and press enter. The Android device should reboot into the bootloader (some device may call it fastboot mode instead).

In command prompt w/adb: type "adb reboot bootloader" + press enter
  • The Android device should now be in the bootloader, and will not be properly recognized by fastboot comands ran on the PC, because of the missing Android Device driver.

  1. Download the Andorid Device Driver files and unzip/extract the files from zip/compressed file. Google Android USB drivers @ https://developer.android.com/studio/run/win-usb

  2. On the PC, open Windows Start menu, type “device manager” and press enter.

Win-key > type "device manager" + press enter = Device Manager menu

  1. On the PC in Device Manager menu, locate and expand “Portable Devices” or “Other Devices”, depending on which is present, and then right-click the name of the device(will be named “Android” or something else generic(i.e. not your device’s actual name)) you connected and select Update Driver Software. Device name is usually just “Android” when the Android device is in the bootloader and connected to the PC.

  2. In the Driver Update wizard, select “Browse my computer for drivers” (2nd option listed).

  3. In the Driver Update wizard, select “Let me pick from a list of available driver from my computer” (2nd option listed).

  4. In the Driver Update wizard, verify that the “Show all devices” is highlight, and click Next(on the bottom right).

  5. In the Driver Update wizard, click “Have Disk” button.

  6. In the “Install From Disk” pop-up click “Browse”, Navigate to the folder where you extracted the Google USB driver, and select/hightlight the driver, filename is android_winusb.inf, and then click “Open” in the lower right corner. Then click “OK”(in upper right corner).

  7. In the Driver Update wizard select/highlight “Android Bootloader Interface” (2nd option in the list), and click “Next”.

  8. A generic warning will pop-up mentioning compatability cannot be verified, but it can be ignored. Continue onto the installation process by clicking “Yes”.

  9. Windows Security will display a pop-up asking if you want to install the driver/device software. Click “Install” to continue the installation process.(You may not get this warning depending on how Windows Security is configured on your PC.)

  10. When the install has completed, the Driver Update wizard will display a message stating the driver has been successfully updated. And its name will be changed from “Android” to “Android Bootloader Interface”.

  11. On the PC, in the command prompt with adb type “fastboot devices” and press enter. The command prompt should output the device id. This device ID should match the device id from the adb devices output, that was in section 2-2.

In command prompt w/adb: type "fastboot devices" + press enter
  • If their is an error or the output is not matching restart the PC, and do subsection 2-2 and section 3 step 2 and this step (15) again. -If the device id matches, then the Android device driver has sucessfully installed and you can close out of the Device Manager.
  1. To reboot the device and exit the boatloader/fastboot mode on the device, on the PC in the command prompt with adb type “fastboot reboot” and press enter.
In command prompt w/adb: type "fastboot reboot" + press enter

4. Verifing ADB and Fastboot function

  • In this subsection you will only be using both the PC and the Android device.
  1. Check ADB works and is up-to-date on your PC. In the command prompt with adb or terminal type and press enter:
adb version
  • If the command fais to run close the command prompt or terminal. Check sections 1-1, 1-2, and/or 1-3 for any steps you may have missed or misconfigured.
  1. Check ADB works with your device. With the device connnected to the PC and the screen turned on and on the homescreem, in the command prompt with adb or terminal type and press enter:
adb devices
  • Save/write down the device number that is shown for reference in followiing steps. -On the device, allow the USB debugging by accepting your PC’s RSA key fingerprint if prompted too, and rerun the adb devices command in the command prompt with adb or terminal.
  1. Reboot into your devices bootloader/fastboot mode. With the device connnected to the PC and the screen turned on and on the homescreem, in the command prompt with adb or terminal type and press enter:
adb reboot bootloader

-Wait some moments for the command prompt or terminal to process the command and the device to reboot into the bootloader/fastboot mode.

  1. Check ADB works and is up-to-date on your PC. In the command prompt with adb or terminal type and press enter:
fastboot --version
  • If the command fais to run close the command prompt or terminal. Check sections 1-1, 1-2, 1-3, 2-1, 2-2, and/or 3 for any steps you may have missed or misconfigured.
  1. Check Fastboot works with your device. When the device has finished loading into the bootloader, check that the device is still connected to the PC. Android Device screen may appear off or very dim(that depends on the device manafacturer configuration). In the command prompt or terminal type and press enter:
fastboot devices
  • Check that the the device number that is shown matches the device number you saved/wrote down in step 2.
  1. If the device number matches that means ADB and fastboot are functioning successfully. In the command prompt or terminal type and press enter:
fastboot reboot
  • If the device number does not match or does not appear, manually reboot the phone by holding the power button untill the device turns off and then turn the device back on. Check sections 2-1, 2-2, and/or 3 for any steps you may have missed or misconfigured.
