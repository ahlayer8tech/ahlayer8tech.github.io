Introduction

This guide intends to provide a step-by-step guide with pictures of how to publish a GitHub Pages website and create the corresponding repository for it. Some familitarity of Github and the markdown syntax language is advised before following this guide.

Github Pages uses the JAMstack web architecture to host websites. GitHub Pages only works with a static website generators, such as Jekyll, Gatsby, Hugo, Next.js, Eleventy, etc… This guide does NOT go over Githubs built-in Jekyl static website builder, because I do find their themes intresting for the category of technology blogs, and nor configuring custom urls for GitHub Pages. Familiarity with the JAMstack and static website generators is highly recoemend before proceeding with the steps of this guide.

1. Login onto GitHub

  1. Go to github.com/login

  2. Input your credntials to sign

   

2. Creating a repository for the site

  1. In the top-right corner of the page click the ➕⬇️ to show the drop-down menu.

  2. In that drop-down menu click “New repository”, at the top of that drop down menu.

  • You will be taken to the “Create a repository page”

  1. In the “Owner” field, verify the new repository will be created under you or your organizations ownership, click the drop-down menu and click the username that you want.

  2. In the “Repository name” field, next to te owner field, enter yourusernamehere.github.io or yourorganizationsnamehere.github.io.

  • If your user or organization name contains uppercase letters, you must use lowercase the letters.

  1. Enter a brief and precise description for your or your organizations for clarity purposes.

  2. Choose a repository visibility for the repository.

  • Repository must be “Public” for you to be able to make a GitHub Pages site.

  1. Select/checkmark “Add a README file” option in the “Initialize this repository with:” section.

  2. Scroll to the bottom of the page, and click “Create repository”, and wait for GitHub to create the repository.

  3. When the repository has finished being created, Github will take you to the repository’s main page.

   

3. Creating the test site

  • You can only create one user or organization site for each account on GitHub. Project sites, whether owned by an organization or a personal account, are unlimited.
  • There are three types of GitHub Pages sites: project, user, and organization. User and organization sites are connected to a specific account on GitHub.com. Project sites are connected to a specific project hosted on GitHub, example a JavaScript library, a recipe collection, etc…

  1. Once at the main page of the site repository (“Code” tab), click on the “Add file” drop-down menu, next to the green “Code” box drop-down menu.

  2. In that drop-down menu click “Create new file”, at the top of that drop down menu.

  3. Name the file “index.md”, and then write brief yet precise description for clarity. -Verify that the file will be created/commited in the “main” branch.

  4. Scroll to the bottom of the page, and click the green button that says “Commint new file”, and wait for GitHub to create the repository.

  • Best practice is to give a title and description of changes when adding or changing files on Github.
  • Verify that the files are being commited directly to the “main” branch.

  1. When the GitHub has finished and reloaded back into the main page for the site repository (“Code” tab), click on the “Settings” tab, at the top left.

  2. In the “Code and automation” section of the sidebar in the repository settings menu, click “Pages”.

  3. In the “GitHub Pages” menu page in the source section click on the “None⬇️” field.

  4. In that drop-down menu click “main”, at the top of that drop down menu.

  • You are changing the default source branch from “None”

  1. In the “GitHub Pages” menu page in the source section click the Save button, on the far left.

  2. Visit your GitHub Pages website to verify the initial test index.md file displays.

  • It can take up to 10 minutes for changes to your site to publish after you save/push the changes to GitHub.
  1. You can check your deployment of the GitHub Pages website from the main page of the repository(“Code” Tab) at the lower left hand corner of Github.

   

Congratulations, ⭐, on creating the foundations for your own custom website that will use GitHub Pages. It is now time upload your website files.

   

4. Uploading your website files and folders

  1. At the main page of the site repository (“Code” tab), click on the “Add file” drop-down menu, next to the green “Code” box drop-down menu.

  2. In that drop-down menu click “upload files”, at the top of that drop down menu.

  • You will be taken to the “Upload” page for the site repository.

  1. Once at the upload page for the site repository, click and drag all your website files from the folder on your PC to the box that says “Drag files to add them to your repository” (it is bolded text).

  2. After you have clicked and dragged all your files to the folder, you have to wait for all those files to upload to GitHub.

  • A green progress bar will display under the “Drag files to add them to your repository” box will show the prgress of the files uploading. The “Uploading X of X files” text under the green progress bar will show the count of the files that have sucessfully uploaded to Github.
  1. When all the website files have finished uploading. Scroll to the bottom of the page, and click the green button that says “Commit changes”, and wait for GitHub to create the repository. Now wait for GitHub to finish processing the commit.
  • Best practice is to give a title and description of changes when adding or changing files on Github.
  • Verify that the files are being commited directly to the “main” branch.

  1. Verify that all your website files and folders you have uploaded to the site repository.

  2. You can check your deployment of the GitHub Pages website from the main page of the repository(“Code” Tab) at the lower left hand corner of Github.

   

Congratulations, ⭐, on publishing your own custom website using GitHub Pages. It is now time share your cool new website with those you want to share it with.
