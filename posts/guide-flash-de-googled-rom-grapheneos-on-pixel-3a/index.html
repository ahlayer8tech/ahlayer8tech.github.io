  • In this subsection you will only be using the device.

  1. In the “Developer options” menu go to enable Android debugging.

2. Install Android Device Driver - Windows Only - Manually

  • In this section you will be using the PC and the Android device.
  • This section is only required for Windows PCs. The Linux Android device driver comes preinstalled on Linux systems(it is apart of/in the Linux kernel).
  1. Power off/shutdown the Android device. To boot the Android device into the bootloader/fastboot/mode. While the phone is powered off, press and hold the volume down button and power button at the same time, and release the buttons when the phone vibrates and the screen loads into the bootloader/fastboot mode.
volume down button long press + power button long press + at the same time = boot device into bootloader/fastboot mode
  1. Connect the device to your PC via USB charging cable.
  • Your PC will not recognize the device, and that is OK.
  • The follwoing steps wil install the Android device driver that makes it so your PC will recognize the android device while it is in bootloader/fastboot mode.

  1. Download the Andorid Device Driver files and unzip/extract the files from zip/compressed file. Google Android USB drivers @ https://developer.android.com/studio/run/win-usb

  2. On the PC, open Windows Start menu, type “device manager” and press enter.

Win-key > type "device manager" + press enter = Device Manager menu

  1. On the PC in Device Manager menu, locate and expand “Portable Devices” or “Other Devices”, depending on which is present, and then right-click the name of the device(will be named “Android” or something else generic(i.e. not your device’s actual name)) you connected and select Update Driver Software. Device name is usually just “Android” when the Android device is in the bootloader and connected to the PC.

  2. In the Driver Update wizard, select “Browse my computer for drivers” (2nd option listed).

  3. In the Driver Update wizard, select “Let me pick from a list of available driver from my computer” (2nd option listed).

  4. In the Driver Update wizard, verify that the “Show all devices” is highlight, and click Next(on the bottom right).

  5. In the Driver Update wizard, click “Have Disk” button.

  6. In the “Install From Disk” pop-up click “Browse”, Navigate to the folder where you extracted the Google USB driver, and select/hightlight the driver, filename is android_winusb.inf, and then click “Open” in the lower right corner. Then click “OK”(in upper right corner).

  7. In the Driver Update wizard select/highlight “Android Bootloader Interface” (2nd option in the list), and click “Next”.

  8. A generic warning will pop-up mentioning compatability cannot be verified, but it can be ignored. Continue onto the installation process by clicking “Yes”.

  9. Windows Security will display a pop-up asking if you want to install the driver/device software. Click “Install” to continue the installation process.(You may not get this warning depending on how Windows Security is configured on your PC.)

  10. When the install has completed, the Driver Update wizard will display a message stating the driver has been successfully updated. And its name will be changed from “Android” to “Android Bootloader Interface”.

3. Install Android Platform Tools - Linux Only

  • In this section you will only be using the PC.
  • This section is only required for Linux PCs.
  • On Linux PCs USB devices cannot be used as non-root without udev rules being setup for each type of device. The Linux Android device driver comes preinstalled on Linux systems(it is apart of/in the Linux kernel).
  • Do not use the adb files from the Ubuntu and Debian package repositories because they will be very outdated, and this may cause errors due to compatability issues with some newer versions of Android.
  • As of publication of this guide the packages that will be downloaded are on version 28.0.2+6. Link to check the most recent version https://pkgs.org/download/android-sdk-platform-tools-common
  1. Open a terminal and type:
  • For Debian, Ubuntu, LinuxMint, & derivatives:
sudo apt update
sudo apt install android-sdk-platform-tools-common
  • For Fedora & derivatives:
sudo dnf install android-tools
  • For Arch, Manjaro, & derivatives:
sudo pacman -Sy
sudo pacman -S android-udev
  • The proper tools and settings have been installed and configured after the install has completed.
  1. Logout of your account, and then log back into your account. The udev rules should now have been initialized on the Linux PC.

4. Install GrapheneOS Image onto the Android device - Web Installer

  • In this section you will be using both the PC and the device.
  • Firefox and Firefox-based browsers do not support the WebUSB protocol, so those browser will NOT work for the process of installing GrapheneOS onto the device.
  • Confirm you have completed the steps in the preceeding sections, or else the proceeding sections commands will NOT function.
  1. Connect the device to your PC via USB charging cable, or if the cable is already connected verify it is securely inserted on both ends. If you have not already, power off/shutdown the Android device, and boot the Android device into the bootloader/fastboot mode.While the phone is powered off, press and hold the volume down button and power button at the same time, and release the buttons when the phone vibrates and the screen loads into the bootloader/fastboot mode.
volume down button long press + power button long press + at the same time = boot device into bootloader/fastboot mode
  1. On the PC, Open a chrome based browser and go to grapheneos.org/install/web.
  • This guide will be using Brave browser for the install process.
  • Do NOT use Incognito or other private browsing modes on the web browser. There will not be enough space for the browser to save the factory image file.
  • Firefox and Firefox-based browsers do not support the WebUSB protocol, so those browser will NOT work for these proceeding steps.

  1. On the PC, proceed to unlock the device bootloader by clicking the “Unlock Bootloader” button on the webpage. grapheneos.org/install/web#unlocking-the-bootloader

  2. On the PC, the web browser will display a prompt, in the upper left corner, asking you to slect and confirm the device you would like the web browser to connect to.

  3. On the device, select the option to unlock the bootloader. Press volume down button to select UNLOCK THE BOOTLOADER option, and then press the power button to confirm that option.

volume down button press -> UNLOCK THE BOOTLOADER + power button press = confirmation to allow unlocking of device bootloader
  • It will take some moments for the bootloader to unlock.
  1. On the PC, proceed to download the image file by clicking the “Download release” button on the webpage. grapheneos.org/install/web#obtaining-factory-images
  • The browser will automatically dectect the android device connnected to your PC and download the appropriate image file.
  • The webpage will show the download progress under the “Download release”, and show the message “Downloading exampledevicename-exampledate.zip release”.
  • It will take some time for the image file to download.

  1. On the PC, the image file download will be completed when the download status bar under the “Download release” button on the webpage says “Downloaded exampledevicename-exampledate.zip release”.

  2. On the PC, proceed to install the image file by clicking the “Flash release” button on the webpage. grapheneos.org/install/web#flashing-factory-images

  • On the device, while the installation is happening, the device will reboot serveral times and the screen will flash and change to several different menus. DO NOT touch the phone during this time.
  • On the PC in the browser, the image file is being installed on the device. DO NOT do anything to that command prompt, let it run. The image install will take awhile to install.
  • The image install process will take awhile.

  1. On the PC, the image file install will be completed when the install status bar, under the “Flash Release” button, is repelaced with the message with “Flashedexampledevicename-exampledate.zip to device”.

  2. On the PC, proceed to unlock the device bootloader by clicking the “Lock Bootloader” button on the webpage. grapheneos.org/install/web#locking-the-bootloader

  3. On the device, select the option to lock the bootloader. Press volume down button to select LOCK THE BOOTLOADER option, and then press the power button to confirm that option.

volume down button press -> LOCK THE BOOTLOADER + power button press = confirmation to allow locking of device bootloader
  • It will take some moments for the bootloader to lock.
  1. If no errors have popped-up during the the install process then that means GrapheneOS has been successfully completed. Press the power button with the default Start option selected in the bootloader menu to boot into the OS. Proceed to the next section. grapheneos.org/install/web#booting

5-2 Encrypt the Device

  • On newer versions of the CalyxOS image files the phone will encrypt itself on the initial setup of the device. If your device is already encrypted when you check encryption status, you do not need to proceed to this section.

  • In this section you will only be using the device, and will need to plug it into the charger.

  • If you have not already set a passcode for your device, one will be setup one in this section. A encrypted device without a passcode is just silly.

  • We will be doing a full-disk encryption as that is the most pravcy protecting encryption type for a device.

  • Warning: Disruptions during the encryption process may cause errors or raise issues. Disconnect and turn off the WiFi and set the device to airplane mode to avoid those issues.

  • Warning: If you are prone to loosing you device and do have have a consistent back-up policy it is not advised to do full-disk encryption on the device because Find-My-Device services will not work if the device is turned off.

  • Warning: Alarms - When the the device is turned off and has full disc encryption, Alarms will not work, because the Clock app will not be able to decrypt the Android Device.

  1. Plugin your charging cable to the device and a power source. The device battery needs to be at or above 80% full before you begin.

  2. On the device, In the “Developer Options” set “Root Access” to “ADB only”. Note: If you do not set the device’s “Root Access” to “ADB only” the rest of these following steps will not work.

Settings > Security > Encryption > Encrypt phone

5-4 Download Your Apps

GrapheneOS is comes with a very minimal amount of apps. It is recomended to download and install F-Droid appstore (Free and opensource only apps), and if you want to download any non-FOSS apps, you will need to download Aurora Store (a private Google Play appstore frontend). Amazon Appstore is an option if you need another appstore. Alternatively, you can always download the specific individual app apk from APK Mirror, and install the app apk manually on to the device.
